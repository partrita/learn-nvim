# Vim 언어 이해하기

Vim의 가장 큰 매력은 단순히 단축키를 외우는 게 아니라, 마치 언어처럼 명령어를 조합해서 편집기와 '대화'할 수 있다는 점입니다. 이 원리만 이해하면 복잡한 작업도 직관적으로 처리할 수 있습니다.

## 버퍼, 창(Window) 및 탭
먼저 Vim에서 화면과 파일을 관리하는 세 가지 핵심 개념을 알아보겠습니다.
* **버퍼 (Buffer)**: 메모리에 로드된 파일 그 자체입니다. 하나의 버퍼를 여러 창에서 동시에 열어 편집할 수 있습니다.
* **창 (Window)**: 화면상의 분할된 영역입니다. 자유롭게 이동하거나 크기를 조절하며 여러 파일을 동시에 띄워둘 수 있습니다.
* **탭 (Tab)**: 여러 창을 모아둔 '화면 레이아웃'의 집합입니다. 브라우저의 탭과는 조금 다른 개념이며, 보통은 창 분할(Split)만으로도 충분해서 아주 자주 쓰이지는 않습니다.

##### 유용한 명령어
* `:e <파일명>`: 새로운 파일(버퍼)을 엽니다.
* `:b <버퍼명>`: 열려 있는 다른 버퍼로 빠르게 전환합니다.
* `:vsplit`: 화면을 세로로 나눕니다.
* `:split`: 화면을 가로로 나눕니다.
* 탭에 대해 더 궁금하시다면 `:help tab-page`를 입력해 보세요.

## 명령어 조합의 원리
Vim에서는 `ggyG` 같은 생소한 입력을 자주 보게 됩니다. 이건 파일 전체 내용을 복사하는 명령어인데, 사실 하나씩 뜯어보면 아주 단순한 논리로 구성되어 있습니다.
Vim 명령어는 각 단계가 하나의 '단어'처럼 동작하며 차례대로 실행됩니다.
* `gg`: 파일의 맨 처음으로 이동
* `y`: 복사(Yank) 시작
* `G` (`Shift+g`): 파일의 맨 끝까지 범위를 지정하며 복사 완료

Vim은 명령어를 완성하기 위해 키를 동시에 누르고 있을 필요가 없습니다. 순서대로 톡톡 입력하면 됩니다. 입력 사이의 대기 시간은 `timeoutlen` 설정으로 조절할 수 있습니다.

### 단축키의 의미 (Mnemonics)
Vim의 단축키는 대부분 영단어의 첫 글자를 따왔습니다. 억지로 외우려 하기보다 단어의 의미를 떠올려 보세요.
* `y`: **y**ank (복사)
* `p`: **p**aste (붙여넣기)

앞으로 새로운 기능을 소개할 때마다 연관된 영단어를 함께 적어두겠습니다. 훨씬 기억하기 쉬우실 거예요.

#### 특수 키 표기법
도움말이나 설정 파일에서 자주 보게 될 표기법들입니다.
* `<cmd>`: 명령어 입력의 시작인 `:`와 같습니다.
* `<cr>`: **C**arriage **R**eturn, 즉 `Enter` 키입니다.
* `<Esc>`: `Escape` 키입니다.
* `<C-x>`: `Ctrl + x` 조합입니다.
* `<M-x>`: `Alt + x` (또는 Option + x) 조합입니다.
* `<leader>`: 사용자가 직접 정의해서 사용하는 접두사 키입니다.

더 자세한 정보는 `:help <키이름>`(예: `:help <cr>`)으로 언제든 확인하실 수 있습니다.

###### 리더(Leader) 키 활용하기
Vim은 키보드의 거의 모든 키를 기본 기능으로 사용합니다. 그래서 사용자만의 커스텀 단축키를 만들 때는 접두사 역할을 하는 `<leader>` 키를 앞에 붙입니다.
기본값은 `,`이지만, 대다수 Neovim 사용자는 가장 누르기 편한 `<Space>`(스페이스바)를 리더 키로 설정해 사용하곤 합니다.

## 강력한 도움말 기능 (Help)
Vim 내부에는 방대한 양의 문서가 내장되어 있습니다. 리눅스의 `man` 페이지와 비슷하지만, 편집기 안에서 바로 검색하고 하이퍼링크로 이동할 수 있어 훨씬 강력합니다. 모르는 게 생기면 구글 검색보다 먼저 `:help`를 활용하는 습관을 들여보세요. 성장에 큰 도움이 됩니다.

## 핵심 모드 (Modes)
Vim의 핵심은 '모드'를 전환하며 작업하는 방식에 있습니다. 각 모드의 역할만 명확히 알아도 절반은 성공입니다.
어떤 모드에서든 `Esc`를 누르면 기본 상태인 '일반 모드'로 돌아옵니다. `Esc`는 매우 자주 쓰이므로, 왼쪽 새끼손가락이 닿기 편한 `Caps Lock` 키를 `Esc`로 바꿔 사용하는 걸 강력히 추천합니다.

* **일반 (Normal)**: 가장 기본이 되는 모드입니다. 이곳에서 이동, 복사, 삭제 등의 모든 명령이 시작됩니다.
* **입력 (Insert)** (`i`): 텍스트를 직접 입력하는 모드입니다. 입력이 끝나면 즉시 일반 모드로 돌아오는 게 Vim의 올바른 습관입니다.
* **비주얼 (Visual)** (`v`): 텍스트를 블록 단위로 선택합니다. 선택 영역을 복사하거나 한꺼번에 수정할 때 씁니다.
* **비주얼 라인 (Visual Line)** (`V`): 줄 단위로 텍스트를 선택합니다.
* **명령 (Command)** (`:`, `/`): 하단 바에 명령어를 입력해 파일을 저장하거나, 검색하거나, 설정을 바꿀 때 사용합니다.

### Caps Lock을 Esc로 매핑하기
* **macOS**: 시스템 설정 -> 키보드 -> 키보드 단축키 -> 보조 키에서 바로 변경할 수 있습니다.
* **Linux (GNOME)**: `gnome-tweaks` 도구의 '키보드 및 마우스' 설정에서 변경 가능합니다.
* **Windows**: [AutoHotKey](https://www.autohotkey.com/)를 설치해 `Capslock::Esc` 스크립트를 사용하거나 PowerToys를 활용합니다.
