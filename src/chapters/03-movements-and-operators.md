# 이동과 연산자
Vim의 편집 방식은 크게 '이동(Motion)'과 '연산(Operator)'이라는 두 가지 개념의 조합으로 이루어집니다. 이 원리만 이해하면 복잡한 텍스트 작업도 마법처럼 간단하게 처리할 수 있습니다.

## 기본 동작 알아보기
* `i`: **I**nsert (입력) 모드로 전환합니다.
* `a`: 커서 뒤(**A**fter)에서 입력을 시작합니다.
* `I`: 줄의 맨 처음에서 입력을 시작합니다.
* `A`: 줄의 맨 끝에서 입력을 시작합니다.
* `o`: 아래에 새 줄을 만들고 입력을 시작합니다.
* `O`: 위에 새 줄을 만들고 입력을 시작합니다.
* `u`: 실행 취소(**U**ndo)를 수행합니다.
* `Ctrl + r`: 다시 실행(**R**edo)을 수행합니다.
* `zz`: 현재 줄을 화면 중앙으로 정렬합니다. 가장 자주 쓰이는 화면 제어 기능입니다.

## 효율적인 이동 (Navigation)
#### hjkl 키에 익숙해지기
Vim에서는 화살표 키 대신 `hjkl`을 사용합니다. 처음에는 어색할 수 있지만, 손을 키보드 중심에서 움직이지 않아도 되기 때문에 숙달되면 비교할 수 없을 만큼 효율적입니다.
* `h`: 왼쪽
* `j`: 아래
* `k`: 위
* `l`: 오른쪽

익숙해질 때까지 화살표 키를 아예 비활성화하는 것도 좋은 방법입니다. [vim-be-good](https://github.com/ThePrimeagen/vim-be-good) 같은 게임으로 연습해 보시는 걸 추천합니다.

### 이동 반복하기
이동 명령어 앞에 숫자를 붙이면 그 횟수만큼 동작을 반복합니다. 예를 들어 `3j`는 아래로 세 줄을 한꺼번에 내려갑니다.

#### 단어 단위 이동
* `w`: 다음 단어(**W**ord)의 시작 부분으로 이동합니다.
* `b`: 이전 단어(**B**ackwards)의 시작 부분으로 이동합니다.
* `e`: 현재 또는 다음 단어의 끝(**E**nd)으로 이동합니다.
* `ge`: 이전 단어의 끝으로 이동합니다.

알파벳을 대문자로 입력(`W`, `B`, `E`)하면 공백을 기준으로 하는 더 큰 단위(`WORD`)로 이동합니다.

#### 주요 점프 기능
* `0`: 줄의 맨 앞으로 이동합니다.
* `$`: 줄의 맨 끝으로 이동합니다.
* `Ctrl + u`: 화면을 반 페이지 위(**U**p)로 올립니다.
* `Ctrl + d`: 화면을 반 페이지 아래(**D**own)로 내립니다.
* `%`: 괄호나 따옴표의 짝을 찾아 점프합니다.
* `Ctrl + o`: 이전에 머물렀던 위치로 되돌아갑니다. (여러 번 가능)
* `Ctrl + i`: 다시 다음 위치로 진행합니다.

#### 구조적 이동 (Pair 기반)
`[` 키와 `]` 키를 조합해 코드의 특정 구조 사이를 빠르게 오갈 수 있습니다.
* `]m` / `[m`: 다음/이전 메서드(**M**ethod)로 이동합니다.
* `]]` / `[[`: 섹션 단위로 이동합니다.
* `}` / `{`: 문단(빈 줄 기준) 단위로 이동합니다.
* `]c` / `[c`: 코드 변경 사항(**C**hange) 사이를 이동합니다. (diff 확인 시 유용)

더 다양한 이동 기능을 제공하는 [vim-unimpaired](https://github.com/tpope/vim-unimpaired) 플러그인도 확인해 보세요.

## 검색을 통한 이동
### 파일 내 검색
Vim의 검색은 단순히 단어를 찾는 것을 넘어, 가장 강력한 이동 수단이 됩니다.
* `/`: 정방향 검색을 시작합니다.
* `?`: 역방향 검색을 시작합니다.
* `n`: 다음 검색 결과로 이동합니다. (**N**ext)
* `N`: 이전 검색 결과로 이동합니다.
* `*`: 현재 커서가 위치한 단어를 정방향으로 검색합니다.
* `#`: 현재 단어를 역방향으로 검색합니다.

검색 후 결과가 항상 화면 중앙에 오도록 설정하면 눈의 피로를 줄일 수 있습니다.
```lua
vim.keymap.set('n', 'n', 'nzz') -- 검색 결과 이동 시 자동 중앙 정렬
vim.keymap.set('n', 'N', 'Nzz')
```

### 한 줄 내에서의 빠른 점프
현재 줄 안에서 특정 문자를 찾아 빠르게 이동합니다.
* `f`: 입력한 문자가 나타나는 다음 위치로 점프합니다. (**F**ind)
* `F`: 이전 방향으로 해당 문자를 찾습니다.
* `t`: 입력한 문자 바로 앞(**T**ill)까지 점프합니다.
* `T`: 이전 방향으로 문자 바로 뒤까지 점프합니다.

줄 내부에서 텍스트를 수정할 때 매우 유용하니 꼭 익혀두세요.

## 연산자 (Operators)
연산자는 단독으로 쓰이기보다 이동 명령어와 결합할 때 진정한 위력을 발휘합니다.
* `y`: 복사 (**Y**ank)
* `d`: 삭제 (**D**elete)
* `c`: 변경 (**C**hange) - 삭제 후 바로 입력 모드로 전환합니다.
* `<` / `>`: 들여쓰기를 줄이거나 늘립니다.
* `=`: 코드를 자동으로 정렬합니다.
* `gc`: 선택 영역을 주석 처리하거나 해제합니다. ([Comment.nvim](https://github.com/numToStr/Comment.nvim) 필요)

연산자를 대문자로 입력(`D`, `C`, `Y`)하면 현재 커서부터 줄 끝까지 적용됩니다. 또한 같은 키를 두 번 연속으로 누르면 현재 줄 전체에 적용됩니다. (예: `yy`는 한 줄 복사)

## 완성된 액션 = 연산자 + 이동
Vim의 정수는 바로 이 조합에 있습니다. 연산자와 이동을 결합해 하나의 완성된 '액션'을 만듭니다.
* `y3k`: 현재 줄을 포함해 위로 세 줄을 복사합니다.
* `ct,`: 현재 위치부터 쉼표(`,`)가 나오기 전까지 내용을 삭제하고 입력을 시작합니다. 함수 매개변수를 수정할 때 특히 유용합니다.

이런 조합을 자유자재로 쓰려면 이전에 설정했던 `relativenumber`(상대 줄 번호)가 큰 힘이 됩니다.

## 이 많은 단축키를 어떻게 다 기억하나요?
단축키를 무작정 암기하려 하지 마세요. 대신 편집기와 나누는 **'대화'**라고 생각해보는 겁니다.

예를 들어 `y3j`를 입력할 때 머릿속으로 **"Yank 3 Down"**(아래로 3줄 복사해줘)이라고 문장을 만드는 식이죠. 명령어를 문법처럼 이해하기 시작하면, 어느 순간 손이 생각보다 먼저 움직이는 놀라운 경험을 하시게 될 겁니다.
